    HipHop Virtual Machine

    • Début de HHVM en 2010 (quand HPHPc est rendu public)
    • Meilleures performance que HPHPc en 2012
    • Et de plus en plus performant !
    • (facebook.com 100% HHVM depuis 2013)

    Sans Machine Virtuelle (HPHPc)

    Avec Machine Virtuelle (HHVM)

    • Code exécuté dans un environement dynamique (VM)
    • Similaire à CLR pour C# ou la JVM de Java
    • Les "hot path" sont compilés en code machine just-in-time
    • HHBC mis en cache dans SQLite (APC/OpCache like)

    Just-in-Time (JIT) ?

    • Compilation des "points chauds" à la volée
    • Le code compilé est rapide
    • MOAR POWAA !
    • Pas forcément, la compilation JIT est coûteuse
    • Compilation x64 uniquement

    • <?hh
class Poney {
  private int $weight;

  public function __construct(int $weight) {
    $this->weight = $weight;
  }

  public function addWeight(int $delta)
  : Poney {
    $this->weight += $delta;

    return $this;
  }

  public function getWeight(): int {
    return $this->weight;
  }
}

    Collections

  • Vector : Liste ordonnée de valeurs du même type
    • $vector = Vector{'pony', 'unicorn'};
  • Map : Valeurs de même type, clés = string ou int
    • $map = Map{'pony' => 5, 'unicorn' = 42};
  • Set : Liste d'int ou string
    • $set = Set{'pony', 'unicorn'};
  • Pair : Paire de valeur de n'importe quel type, immutable
    • $pair = Pair{'jolicode', 42};
